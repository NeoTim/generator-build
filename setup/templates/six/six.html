What makes a good team development environment?

  • Modularizing your code base
    • Eliminates the risk of having merge conflicts.
    • Allows for quick code base navigation.
    • Provides new teamates with a clean and organized intro, which means "Less confusion".
  • Style Guide for you team to follow.
    • Keeping the team consistent with a style guide allows teamates to scan, and help debug your code.
    • Again, here is a great style guide resourceJohn Papa's Angularjs-Style-Guide.
  • An organized development evironment
    • Use task runners like Gulp or Grunt
    • Using task runners provides a rappid development and prototyping.
    • Comming Soon ng-guide will provide you with a sweet task runner set up that will work for your team.
