Understanding the Code Base

Notice the new file structure

We have several new files that may seem sompletely unfamiliar!

No worries, the are not that harmful, and shouldn't stress you out.

The majority of the files help you with you development evironment.

  • Moved Files
    • Moved .bowerrc outside the client directory.
    • Moved .editorconfig outside the client directory.
    • Moved bower.json outside the client directory.
    • Moved package.json outside the client directory.
  • Client
    • .htaccess
      • Povides apache with server access to your application, when you deploy.
    • .jshint
      • Allows task runners to run jshint on your code, to make sure you do not have syntax errors.
      • Also provides options for following a style rule. John Papa's "Style Guide" provides a nice configuration.
      • Don't worry about grabbing the .jshint from John Papa's style guide, It has already been integrated.
    • favicon.ico
      • This is the little Icon you see for different website on you tabs.
      • It is also the Icon you see for different website in your bookmarks.
      • You don't have ot do anything with it, Apache knows how to serve it up for you.
      • If you decide to make your own, it must be 20x20 pixels.
    • robots.txt
      • Allows you to set robot key words for web crallers
  • Root Directory
    • .gitattributes
      • This is set for github attributes
    • .gitignore
      • Anyhting you set in this file will be ignored when you commit and push to github.
      • Commonly ignored files are your build directories, node_modules, & bower components.
      • Don't worry about ignoring anything yet. We will discuss what need to be ignored later.
    • .travis.yml
      • Travis is a code coverage helper tool. It allows you to see your dependencie versions and other stuff.
      • Dont't worry about the travis.yml file quite yet
    • .yo-rc.json
      • Please do not remove this file. This contains important information for me to provide you with necessary informaition
      • If you remove this, you will have to start over!
  • Don't Stress over the amount of unfamiliar files!
    • Even though we don't need half these files yet, It is best to get use to the being in you codebase.
    • Especially if you plan on developing professionally... you will see more files like this that are un familiar.
