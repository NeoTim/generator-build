Currently we have the following

  • A fulling functioning angular application.
  • Grabbing it's dependencies from bower.
  • Pretty much the entire app is inside the client directory.
  • All other script, styles, and view files are in the client/app directory.
  • We have a bunch of other unfamiliar files on the outside.
  • Finally we have a server.js file, serving up our app.

Cool! so What is in our angular application?

I like to think of angular app architecture, as an object.

the directory names are the keys, and contents within is a bunch more objebcts.

This first key to the object is the app.js file.

  • We have a root app object that all other files are connecting to.
  • Considering all the other files are connecting to the root app file... Shouln't the app.js file be the first app file loaded?
